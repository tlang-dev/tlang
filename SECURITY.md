# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| 0.1.x   | Yes       |

## Reporting a Vulnerability

**Please do not open a public GitHub issue for security vulnerabilities.**

Report security issues by email to **joel@tlang.dev**. Include:

- A description of the vulnerability and its potential impact
- Steps to reproduce or a minimal proof-of-concept
- Any relevant versions or configurations affected

You will receive a response within 72 hours acknowledging receipt. We aim to release
a fix within 14 days of a confirmed vulnerability, depending on severity and complexity.

We follow responsible disclosure: once a fix is released, we will publicly disclose
the vulnerability in the release notes with credit to the reporter (unless you prefer
to remain anonymous).
